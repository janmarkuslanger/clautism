$ ollama pull qwen2.5 && ollama serve

# list everyone you'd otherwise forget
$ cp data/people.example.json data/people.json

# acquire a robot mouthpiece
$ export TELEGRAM_BOT_TOKEN="123:ABC..."

# become a wonderful person, instantly

Objections, handled

Frequently rationalized questions

Isn't this deeply impersonal?

Yes. But it's impersonal at scale, which in the right light looks exactly like personal.

What if they find out an AI wrote it?

Then you've successfully transferred a difficult conversation to your descendants. Synergy.

Does it really run 100% locally?

Yes — Ollama on your machine, a flat JSON file, no cloud, no API keys. The only thing leaving your computer is the actual Telegram message and, arguably, your dignity.

Can I add more channels?

Drop one file in birthday_flow/channels/ with a @register_channel class. The core flow never changes. We call this "omnichannel". It is two channels.

Is there a catch?

You still have to type their names into a JSON file once. We're working on an AI for that.
